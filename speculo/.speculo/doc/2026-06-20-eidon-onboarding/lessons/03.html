1. IPC 通道全景

域通道数前缀示例

💡 通道命名规则：<域>:<camelCase动作>。req/res 类型在 IpcContract 中结对声明，编译期 + 启动期双重校验全覆盖。

2. 请求/响应全链路（以「扫描节点树」为例）

用户打开一个 workspace，前端需要扫描所有节点的树结构。全程 9 步：

📍 Step 1 — frontend/stores/nodes.ts
store 调 bridge：nodesBridge.scan(root)
↓
📍 Step 2 — bridge/ipc/nodes.ts
bridge 调 window.eidon：eidonInvoke('nodes:scan', { root })
↓
📍 Step 3 — bridge/ipc/client.ts
eidonInvoke → window.eidon.invoke('nodes:scan', { root })
↓ (Electron IPC 跨进程)
📍 Step 4 — preload/index.ts
contextBridge 转发 → ipcMain.handle('nodes:scan', ...)
↓
📍 Step 5 — backend/ipc/handlers/nodes.handlers.ts
handler 解析参数，调 service：nodeService.scan(root)
↓
📍 Step 6 — backend/services/node-service.ts
编排层：构造 workspace-store（注入端口）→ 调 domain.scan() → 经 shared/contracts 校验
↓
📍 Step 7 — backend/domain/nodes.ts
业务规则：遍历目录 → 识别 .node/ → 深度=层级 → id→path 映射 → 构建 NodeTree
↓
📍 Step 8 — backend/capabilities/editor/file-ops.ts
底层能力：fs.readdir / fs.readFile 读 .node/node.json
↓ (结果沿原路返回)
📍 Step 9 — frontend/stores/nodes.ts
store 收到 NodeTree，更新 Zustand 状态 → React 重新渲染 FileTree

3. 事件推送（反向：main → renderer）

除了请求/响应，主进程也可以主动推送事件给渲染进程。4 个 eidon:* 事件在 shared/ipc/events.ts 定义：

事件触发时机载荷
eidon:menu用户点击原生菜单项{ action: string }
eidon:close-requested用户尝试关闭窗口{}（触发 close-guard）
eidon:file-changed外部程序修改了文件{ path: string }
eidon:index-updated工作区索引重建完成{ root: string }

推送路径：

backend/capabilities/editor/watcher.ts    ← chokidar 检测到文件变化
  → backend/ipc/emit.ts                   ← webContents.send('eidon:file-changed', { path })
    → preload/index.ts                    ← contextBridge 转发
      → bridge/ipc/events.ts              ← listen('eidon:file-changed', callback)
        → frontend/composables/useFileWatcher.ts

4. channels.ts — IPC 单一事实源

看一个真实的通道定义（简化）：

// shared/ipc/channels.ts
export interface IpcContract {
  'nodes:scan':          { req: { root: string };             res: NodeTree }
  'nodes:create':        { req: CreateNodeInput;               res: NodeMutationResult }
  'nodes:promote':       { req: { path: string; templateId: string }; res: NodeMutationResult }
  'editor:readFile':     { req: { path: string };              res: FileReadResult }
  'editor:writeFile':    { req: { path: string; content: string }; res: void }
  'todos:scan':          { req: { nodeRefs: NodeRef[] };       res: AggregatedTodo[] }
  'git:fileHistory':     { req: { filePath: string };          res: CommitMeta[] }

}

5. 新增一个 IPC 通道的 5 步法（核心技能）

假设要新增 shell:getAppVersion：

步骤文件操作
① 声明通道shared/ipc/channels.ts在 IpcContract 加 'shell:getAppVersion': { req: void; res: string }；补全 CHANNEL_PRESENCE
② 实现能力backend/capabilities/或domain/写纯逻辑（如读 app.getVersion()）
③ 编排backend/services/service 方法调用 capability/domain
④ 接线backend/ipc/handlers/shell.handlers.tsipcMain.handle('shell:getAppVersion', ...) → service
⑤ 前端包装bridge/ipc/shell.ts 或新建getVersion: () => eidonInvoke('shell:getAppVersion')
