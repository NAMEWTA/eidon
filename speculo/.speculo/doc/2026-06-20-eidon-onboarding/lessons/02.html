📂 Lesson 02 · 目录结构逐层详解

耗时：~14 分钟  |  先修：Lesson 01（四层架构）

🏆 本课胜利：给定一个功能名（如「节点扫描」），你能列出它涉及的所有文件路径（从 shared → backend → bridge → frontend）。

⚠️ 关键规则：所有 store 只做 UI 缓存，数据源在磁盘。Store 调 bridge（如 nodesBridge.scan()），bridge 调 window.eidon.invoke()。Store 里面没有任何业务逻辑。

3. bridge/ — 契约边界

bridge/ipc/
├── index.ts                 # 统一出口
├── client.ts                # eidonInvoke → window.eidon.invoke
├── events.ts                # 事件订阅
├── nodes.ts                 # nodesBridge: scan/create/promote/...
├── templates.ts             # templatesBridge
├── todos.ts                 # todosBridge
├── consistency.ts           # consistencyBridge
├── git.ts                   # git 网关
├── snapshots.ts             # snapshots 别名
├── dialog.ts                # 文件对话框
├── clipboard.ts             # 剪贴板
├── notification.ts          # 通知桥
├── opener.ts                # 外部打开
├── path.ts                  # 路径工具
└── tauri.ts                 # Tauri API 适配面

bridge 的每个文件是对一类 IPC 通道的前端包装。典型写法：

// bridge/ipc/nodes.ts
import { eidonInvoke } from './client'
export const nodesBridge = {
  scan: (root: string) => eidonInvoke('nodes:scan', { root }),
  create: (input) => eidonInvoke('nodes:create', input),
}
