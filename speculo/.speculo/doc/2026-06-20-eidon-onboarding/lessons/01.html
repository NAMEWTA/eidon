1. EIDON 是什么

EIDON = 本地优先（local-first）的结构化知识 IDE。给本地 Markdown 知识库装上固定三层节点拓扑 + 多模板 schema 的数据地基。技术栈：

层技术位置
前端（纯 UI）React 19 + TypeScript + Tailwind CSS v4 + Zustand v5 + shadcn/uiapp/frontend/
桥接（契约边界）typed IPC 包装器app/bridge/
共享（数据模型）纯 TypeScript（框架无关）app/shared/
后端（main 进程）Electron main（Node）+ 85 条 typed IPCapp/backend/
接缝preload contextBridge 暴露 window.eidonapp/preload/
💡 历史：由 SoloMD 1.x → EIDON 2.0 颠覆性重构；Tauri 2/Rust → Electron 全 TypeScript（ADR-0024）；三层 → 四层（ADR-0025）。当前不背任何向后兼容包袱。

3. backend 内部的四级联

backend 内部不是平的，有严格的单向级联：

ipc/ ──→ services/ ──→ { domain/, capabilities/ }
 ↑                      ↑              ↑
 可用 electron          禁 electron   禁 electron
 对接 preload           组合两者      仅 node:* + 库
  • ipc/ — 唯一可用 electron 的 backend 层（另一处是 shell/），接 IPC 请求，转发给 service
  • services/ — 编排层，组合 domain 业务规则 + capabilities 底层能力（+ emit 事件）
  • domain/ — 纯业务规则，仅经注入端口（来自 shared/models/ 的接口）+ shared。可独立单测
  • capabilities/ — 底层资源，只 node:* + 库（isomorphic-git/chokidar/nspell）+ emit。可独立单测

4. 关键约束速记

约束详情
唯一接缝preload/index.ts → window.eidon，禁止 frontend/bridge 直接 import electron
全局禁止@tauri-apps/*（Tauri 已删）、frontend/bridge 中 import electron
shared 纯洁性禁 react/zustand/electron，纯类型+纯函数
IPC 穷尽性85 通道在 shared/ipc/channels.ts 声明，backend/ipc/register.ts 启动时校验全覆盖
机器强制eslint-plugin-boundaries + no-restricted-imports，CI 拦截

5. 与三层【节点】的区别

「三层」在本项目是双关词，必须显式限定：

  • 【代码】分层（四层） = frontend → bridge → backend + shared（本课内容）
  • 【节点】三层 = L1 → L2 → L3 数据节点层级（Lesson 04 讲）
