本节胜利：学完本节后，你能画出 EIDON 三层【代码】架构图，并说出每一层的职责、模块清单、以及那条 「碰了就炸」的单向依赖铁律。

1. 你在改的到底是什么？

EIDON 是一个 Local-First 结构化知识 IDE（Tauri 2 + React 19 + Rust）。 它的代码被严格分入三层，每一层只能朝一个方向调用——这条规则由 ESLint 机器强制，CI 拦截违规。

2. 三条铁律（碰了就炸）

铁律一：单向依赖

src → core → src-tauri，不可反向。UI 只能 import core/； core/ 不得 import UI（src/）；Rust 不知道 TS 层的存在。 这由 eslint-plugin-boundaries + no-restricted-imports 机器强制执行。

铁律二：core/bridge/ 是访问 Rust 的唯一出口

只有 core/bridge/** 被允许 import @tauri-apps/api 或调用 invoke()。 ESLint allowlist 必须保持为空（0）。前端代码从不直接调 invoke()， 而是通过 bridge 的 typed wrapper——这让调用链可追踪、可类型检查。

铁律三：core/ 全层禁 UI 框架

core/ 下任何文件禁止 import react / zustand 及任何 UI 框架。 这保证核心逻辑可在 Node 下纯函数式单测，不依赖浏览器/DOM。

3. core/ 的四根柱子

EIDON 数据层按细模块切分，新增了四个并列业务模块（工程层 ADR-0012）。 各自有 index.ts、禁 UI 框架、互不 import 对方内部路径，只能经各自 index.ts 暴露的公共 API 通信。

🏗️ nodes

节点身份(ULID) · node.json 读写 · 扫描建树(深度=层级+id→path) · 创建/重命名/移动/提升为节点

📋 templates

6类字段 schema 版本化 · 编辑生成新版 · 删除孤儿模板态 · 内置种子 · 给设置面板供数据

📸 snapshots

版本能力归属：直接复用现有 git（autoGit 不改、不做快照功能），仅薄封装 git bridge

🔍 consistency

结构违规检测+标记（四类）· 只读不阻塞不自改 · 违规在 FileTree 打徽标

⚠️ 关键消歧：「三层」一词在本仓库双关，禁止裸用。 必须显式写 三层【代码】（= src→core→src-tauri，软件分层）或 三层【节点】（= L1/L2/L3，数据节点层级）。两者是完全不同的概念。

同样，「Agent/AGENTS.md」「workspace」「AutoGit/快照」均为多义词，出现时必须标注语境（见 CONTEXT.md §1）。
