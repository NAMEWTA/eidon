🔌 第2课：通道一 — Tauri Command 全链路

目标：完全理解 invoke('read_file', {path}) 从 JS 到 Rust 再返回的每一个环节。

一、什么是 Tauri Command？

一句话：Tauri Command 是前端 JS 调用 Rust 函数的 RPC 机制。
类比：就像前端的 fetch('/api/read-file') → 后端的 app.get('/api/read-file', handler)。
区别在于 Tauri 不走 HTTP，而是通过 IPC（进程间通信）直接调用，零网络开销。

二、完整调用链（以读文件为例）

// ===== 第 1 步：前端发起调用 ===== // app/src/composables/useFiles.ts import { invoke } from '@tauri-apps/api/core'; const result = await invoke('read_file', { path: '/Users/alex/notes/hello.md' }); // result = { content: "# Hello\n...", encoding: "UTF-8", language: "markdown", had_bom: false } // ===== 第 2 步：Tauri IPC 层序列化 ===== // @tauri-apps/api/core 将 { path: "…" } 序列化为 JSON // 通过 WebView IPC bridge 发送给 Rust 端 // ===== 第 3 步：Rust 端反序列化 + 路由 ===== // Tauri 运行时：查找命令名 "read_file" → 找到 file_ops::read_file // 将 JSON 参数反序列化为 Rust String 类型 // ===== 第 4 步：执行 Rust 函数 ===== // app/src-tauri/src/editor/file_ops.rs #[tauri::command] pub async fn read_file(path: String) -> Result<FileReadResult, String> { // 将 CPU 密集型工作（读磁盘）移到线程池 tauri::async_runtime::spawn_blocking(move || read_file_inner(path)) .await .map_err(|e| format!("join: {e}"))? } pub fn read_file_inner(path: String) -> Result<FileReadResult, String> { // 1. 读原始字节 let bytes = fs::read(&path)?; // 2. 检测编码 (UTF-8 / GBK / UTF-16 …) // 3. 解码为 UTF-8 字符串 // 4. 返回 FileReadResult { content, encoding, language, had_bom } Ok(FileReadResult { content, encoding, language, had_bom }) } // ===== 第 5 步：返回序列化 → 前端收到 Promise resolve ===== // Rust 端：FileReadResult → JSON // JS 端：JSON → JS Object { content, encoding, language, had_bom }

三、三个关键机制深入理解

3.1 为什么需要 spawn_blocking？

关键知识点：Tauri 命令默认在主线程（UI 线程）上执行。如果 read_file 是同步的且文件很大（或磁盘慢），整个窗口会冻结直到 I/O 完成。

解决方案：声明为 async fn + 用 spawn_blocking 把实际工作移到后台线程池。

EIDON 中几乎所有涉及文件 I/O、git 操作、PDF 解析的命令都用了这个模式。

3.2 参数自动注入

Tauri 能从 Rust 函数签名中自动识别并注入特殊参数：

参数类型Tauri 注入什么用途
String, usize, Vec<T>, 自定义 struct… 从 JS 传来的 JSON 参数自动反序列化 接收前端传参
app: tauri::AppHandle 应用全局句柄 发事件、获取路径
state: tauri::State<T> 通过 .manage() 注册的共享状态 访问共享数据
window: tauri::Window 调用命令的窗口 操作窗口（关闭、打印）
// 示例：watch_file 命令同时用了 3 种注入 #[tauri::command] pub fn watch_file( app: AppHandle, // ← Tauri 自动注入 state: tauri::State<WatcherState>, // ← Tauri 自动注入 path: String, // ← 前端传参 ) -> Result<(), String> { ... }

3.3 错误处理：Result<T, String>

Tauri Command 的标准返回签名是 Result<T, String>：
• Ok(value) → 前端 await invoke(...) 得到 value
• Err("错误信息".to_string()) → 前端 invoke(...) 抛出异常，catch 到错误字符串
• EIDON 所有命令的错误消息都设计为用户可直接看懂的文本（用于 toast 弹窗）

四、如何新增一个 Command（实战模板）

// ======== 第 1 步：在 Rust 端写函数 ========
// app/src-tauri/src/some_module.rs
use serde::Serialize;

#[derive(Serialize)]              // ← 自动转 JSON 返回前端
pub struct MyResult {
    pub message: String,
    pub count: usize,
}

#[tauri::command]                 // ← 魔法宏
pub async fn my_command(
    input_path: String,           // ← 前端传来的参数
    max_items: usize,
) -> Result<MyResult, String> {   // ← 返回 Result
    tauri::async_runtime::spawn_blocking(move || {
        // 你的业务逻辑…
        Ok(MyResult {
            message: format!("处理了 {}", input_path),
            count: max_items,
        })
    }).await.map_err(|e| format!("join: {e}"))?
}

// ======== 第 2 步：在 lib.rs 或 runner.rs 注册 ========
.invoke_handler(tauri::generate_handler![
    // ... 已有命令 ...
    some_module::my_command,      // ← 加这一行
])

// ======== 第 3 步：前端调用 ========
// app/src/.../YourComponent.ts
import { invoke } from '@tauri-apps/api/core';

const res = await invoke<MyResult>('my_command', {
  inputPath: '/some/path',
  maxItems: 10,
});
⚠️ 命名约定：Rust 函数名用 snake_case（如 my_command），JS 端传参的 key 自动 camelCase（如 inputPath）。Tauri 的 serde 默认会做这个转换！
