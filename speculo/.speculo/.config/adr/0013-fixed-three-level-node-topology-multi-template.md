# ADR-0013 · 固定三层【节点】拓扑 + 深度=层级铁律 + 多模板 schema

**状态：** 已锁定（工程兑现产品层 ADR-002/005；本期数据层核心规范）

> **来源消歧：** 本条是**工程层 ADR**，兑现**产品层** `temp/EIDON_数据层架构决策记录_ADR.md` 的 `ADR-002`（结构=固定三层节点拓扑+多模板）与 `ADR-005`（schema 版本化）。引用产品愿景写「产品层 ADR-002」，引用本条写「工程层 ADR-0013」。

EIDON 数据层的不可动摇前提：

- **结构节点深度固定 3 层。** workspace 根下**第 1/2/3 层节点目录 = L1/L2/L3，第 4 层起为自由文件夹/文件**（无身份、系统不管）。
- **深度=层级铁律：** 层级由物理位置唯一确定，数嵌套深度即可，不需读 `.node/` 推断、无歧义。
- **创建期硬强制（零妥协）：** 通过 EIDON 只能按 `根→L1(选模板) / L1下→L2 / L2下→L3 / L3下→自由` 创建；UI **不存在**任何越级创建入口。L1/L2 为纯组织层（白名单仅 `.node/`+`README.md`+`AGENTS.md`+下级节点，无内容文件）；L3 为唯一内容承载层。
- **多模板并列：** 同一 workspace 可并列多套三层模板，每套自定义三层各自名字 + 字段集；建 L1 时选模板，L2/L3 继承父链。
- **节点 vs 普通文件夹：** 含 `.node/` 子目录者为结构节点，不含者为普通文件夹（产品层 ADR-004）。本期两者共存。
- **schema 二分：** 元字段（id/templateId/level/type/schemaVersion/createdAt/flags，与模板无关、初始化即生成、用户不可增删）+ 扩展字段（模板定义的业务字段，6 类型，可扩展，版本化）。

**本期对产品层愿景的诚实收紧（见 ADR-0016）：** 产品层 ADR-002 / AX-5 靠「自动补全 + 杂物自动整理」兜底前三层完整性；本期改为**创建期硬强制 + 检测期只标记不自动改**。深度=层级铁律、固定三层、多模板这三条**完整兑现**，不打折。

## Consequences

- 深度锁死为 3（产品层 R-1 风险）：第 4 层若需身份要改地基，用户已接受用普通子目录解决。
- 「三层【节点】」必须与「三层【代码】」全程消歧（CONTEXT.md）。
- 节点磁盘形状（node.json + 三层白名单）由 ADR-0014 纳入契约防漂移。
