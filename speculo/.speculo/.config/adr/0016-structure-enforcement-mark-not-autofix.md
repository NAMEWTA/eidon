# ADR-0016 · 结构强制 + 违规标记（检测不自动改写）

**状态：** 已锁定（`core/consistency` 本期范围；收紧产品层 ADR-007/008/011/012 + AX-5）

EIDON 的「结构一致性」本期 = **创建期硬强制（第一道防线） + 检测期只标记不自动改（第二道防线） + 用户点击整改**。`core/consistency` 模块承载第二、三道防线。

- **创建期硬强制（ADR-0013）：** UI 不给越级创建口子、L1/L2 不放内容文件、新建内容文件仅 L3——保证 EIDON 内产生的结构永远合规。
- **检测期标记（只读、不阻塞、不自动改）：** 打开/扫描时 `core/consistency` 检测四类结构违规并在 FileTree 打徽标：① 前三层出现普通文件夹（无 `.node/`）→「待提升/非节点」；② L1/L2 出现内容文件 →「位置非法/待下沉」；③ 节点物理深度与 `level` 不符 →「层级不符」；④ `.node/` 缺失或损坏 →「结构待修复」。
- **整改（用户点击触发，系统绝不自动动）：** 主入口=「提升为节点」（普通文件夹按当前物理深度提升）；其余引导手动移动/下沉/删除/重建 `.node/`。

**对产品层愿景的诚实收紧（驱动力）：** 产品层 AX-5 受限例外允许「自动补全缺失系统文件」、ADR-008 允许「自动整理开关」、ADR-007/011/012 引入 `outOfPlace/orphan/orphanTemplate/disconnected` 软态身份系统 + 独立一致性面板。本期把这些**全部降级**：

| 产品层愿景 | 本期 |
|---|---|
| 自动补全缺失系统文件（AX-5 例外） | 不补全，只标「结构待修复」，用户点「提升/重建」 |
| 杂物自动整理 + 「其他」L3 兜底节点（ADR-008） | 不自动移动，只标「位置非法」，用户点「下沉到 L3」 |
| 软态身份系统 + 独立一致性面板（ADR-007/012） | 无软态 `flags`、无面板，只在 FileTree 标记 |
| 回收站三关恢复（ADR-011） | 复用现有 FileTree 直接删除，无回收站 |

「系统绝不在我背后动我的文件」这个确定性，比省一次点击更重要，与「完全本地、数据可控」立项初衷一致。

## Consequences

- `core/consistency` 本期只产出「检测结果 + 标记」数据，不写 node.json 软态 flag、不移动文件。
- 软态体系 / 自动补全 / 自动整理 / 一致性面板 / 回收站三关 = 后续迭代，在 `core/consistency` 与现有删除之上叠加，不改已立节点+模板模型。
