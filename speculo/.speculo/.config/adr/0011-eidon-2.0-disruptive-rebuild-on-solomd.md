# ADR-0011 · EIDON 2.0：以现有系统为基底的颠覆性产品重构

**状态：** 已锁定（根决策，决定后续一切；对应 decision-log D-1）

EIDON 是基于 SoloMD 的**一次完整的产品迭代重构升级——本质上的 1.x → 2.x 跨越**，而非增量叠加。产品形态从「Agent Recipes 驱动的扁平 Markdown 库」**颠覆性**转向「Local-First 结构化知识 IDE」：以**固定三层【节点】拓扑 + 多模板 schema** 数据层为**新地基**，由它重新收编现有编辑 / 版本 / 搜索 / 文件树等成熟能力。

两条同时成立的硬约束：

1. **不背兼容包袱。** 无需考虑旧 SoloMD 用户数据 / 旧磁盘形态 / 旧品牌的向后兼容（呼应 ADR-0005）。可以放手改名（ADR-0017）、重立系统区（`.eidon/`）、重订磁盘契约（ADR-0014）。
2. **以现有系统为基底，复用优先、最小新增。** 颠覆的是**产品形态与数据模型**，不是**工程实现**。三层【代码】架构（ADR-0006/0007）与 CodeMirror 编辑栈、git 版本能力、全文搜索、文件树等基础设施**整体复用**；新地基以「新增业务模块 + 改造 FileTree」的既定方式落地（ADR-0012）。**典型示例：版本 / diff 直接用现有 git（autoGit），autoGit 无需修改、不另做快照功能（ADR-0015）。**

## Consequences

- 「重构」不等于「推倒重写」：地基是新的（节点 + 模板），绝大部分既有能力被**复用**，而非重造。凡能复用现有实现的，一律复用、不新建平行功能。
- AI·Agent·Recipes 不在 EIDON 产品范围内：其现有代码作为基底保留在仓库，但 EIDON 不挂载、不依赖；未来若做 AI，是一次**独立重建**（详见 ADR-0018，不在本期范围）。
- 「三层」一词双关必须全程消歧（三层【代码】vs 三层【节点】），见 `context/CONTEXT.md`。
