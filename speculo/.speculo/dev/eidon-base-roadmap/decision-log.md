## 本期范围边界（IN / REUSE / OUT 一览）

**IN（本期必造）：** `.eidon/templates/` 多模板版本化 schema（6 类字段，设置内管理）· 节点 `.node/node.json`（id/templateId/level/type/schemaVersion/fields）· 创建期硬强制 depth=层级 · L1/L2 纯组织层 / L3 唯一内容层 · 打开扫描建节点树（id→path）· 节点 CRUD（创建/重命名 ID 不变/移动）· 提升为节点 · 节点字段表单 · 模板管理 UI（设置内）· 节点感知 FileTree + 违规标记 · 删缓存 100% 从文件重建 · solomd→EIDON 改名 · 磁盘契约统一纳入 zod+fixtures。

**REUSE（本期复用现有，收编进新地基）：** 编辑/预览/大纲（CodeMirror 栈）· 失焦自动保存 · 版本/diff（直接用现有 git，autoGit 不改、不做快照功能；`core/snapshots` 仅薄封装）· 外部修改刷新（watcher）· 全文搜索（GlobalSearch）· 删除（FileTree 直接删除）· 已存在且有价值的能力（可超前实现）。

**OUT（本期不做，后续迭代）：** 私有 `.eidon/snapshots.git` 等「另造快照系统」（**用现有 git 即可**）· 回收站三关（`.eidon/trash`）· 软态身份系统（outOfPlace/orphan/disconnected）+ 自动补全 + 自动整理 + 独立一致性面板 · 用户层链接 `[[]]`/块引用/反向链接（除非已存在可超前）· 语义/向量搜索 · **AI·Agent·Recipes 整块（不在 EIDON 范围，旧代码作基底）**。

**零成本前向兼容（保留不实现）：** `node.json` 的 `references:[]`、`flags:{}`、节点内 `AGENTS.md` 占位——为后续迭代预留，本期不读不写业务逻辑。
